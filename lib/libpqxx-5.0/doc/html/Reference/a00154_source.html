403 [1] Scott Meyers, in one of his essential books, "Effective C++" and "More
404 Effective C++", points out that it is good style to have any class containing
405 a member of pointer type define a destructor--just to show that it knows what it
406 is doing with the pointer. This helps prevent nasty memory leak / double
407 deletion bugs typically resulting from programmers' omission to deal with such
408 issues in their destructors.
409 
410 The @c -Weffc++ option in gcc generates warnings for noncompliance with Scott's
411 style guidelines, and hence necessitates the definition of this destructor,
412 trivial as it may be.
